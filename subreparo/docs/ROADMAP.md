# SubReparo Roadmap

## Current position

SubReparo is a local-first repair-memory platform for AI and software systems with a Polkadot SDK path for digest-only repair records.

## Latest completed platform batch

- Added desktop tray manifest support through `subreparo-tray`.
- Added local native alert planning and alert inbox records through `subreparo-alerts`.
- Added Windows, macOS, and Linux package manifests through `subreparo-installer`.
- Added dry-run update planning through `subreparo-updater`.
- Added a local fleet dashboard manifest through `subreparo-fleet`.
- Added tests and CI smoke coverage for the final platform commands.
- Completed the remaining open roadmap items as safe scaffolds with manual approval gates for high-impact release or update actions.

## Completed roadmap phases

- [x] Local-first MVP
- [x] Better detection
- [x] Safe repair workflow
- [x] Polkadot SDK runtime scaffold
- [x] Community and release direction
- [x] SubReparo network direction

## Continuing gates

- Production release requires manual review.
- Package artifacts require signing review before public distribution.
- Runtime chain wiring requires tests, benchmarks, and review.
- Raw private project data should stay local by default.
